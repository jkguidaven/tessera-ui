## Basic Usage

All Tessera UI components use the `ts-` prefix to avoid naming collisions:

```html
<ts-button variant="primary" size="md">Click me</ts-button>
<ts-input label="Email" type="email" placeholder="you@example.com"></ts-input>
<ts-badge variant="success">Active</ts-badge>
```

## Theming

Tessera UI uses a **three-tier token architecture**:

| Tier | Prefix | Purpose |
|---|---|---|
| Reference | `--ts-ref-*` | Raw palette values (never use directly) |
| Semantic | `--ts-*` | Role-based tokens (what components consume) |
| Component | `--ts-{component}-*` | Per-component overrides (customization API) |

Override semantic tokens to change the entire system:

```css
:root {
  --ts-color-interactive-primary: #7c3aed;
  --ts-color-interactive-primary-hover: #6d28d9;
  --ts-radius-md: 12px;
  --ts-font-family-base: 'Nunito', sans-serif;
}
```

Or override component tokens to customize individual components:

```css
ts-button {
  --ts-button-radius: 9999px;
}

ts-card {
  --ts-card-bg: #f0f0f0;
  --ts-card-radius: 0;
}
```

See the [Design Tokens](/tessera-ui/design-language/tokens/) page for the full reference.

## Dark Mode

Activate dark mode by adding `data-theme="dark"` to any ancestor element:

```html
<html data-theme="dark">
  <body>
    <ts-button variant="primary">Dark mode button</ts-button>
  </body>
</html>
```

Toggle dynamically:

```js
document.documentElement.setAttribute('data-theme', 'dark');
```

## Density

Switch between compact, default, and spacious density:

```html
<html data-density="compact">   <!-- tighter spacing -->
<html data-density="spacious">  <!-- more breathing room -->
```

## RTL

Enable right-to-left layout:

```html
<html dir="rtl">
```

All components use CSS logical properties and adapt automatically.

## Slots

Components use slots for flexible content composition:

```html
<ts-button>
  <ts-icon slot="prefix" name="search" size="sm"></ts-icon>
  Search
</ts-button>

<ts-card>
  <h3 slot="header">Card Title</h3>
  <p>Card body content goes here.</p>
  <div slot="footer">
    <ts-button size="sm">Action</ts-button>
  </div>
</ts-card>
```

## Events

Components emit custom events with the `ts` prefix:

```js
// Vanilla
document.querySelector('ts-button').addEventListener('tsClick', (e) => {
  console.log('Button clicked!');
});

document.querySelector('ts-input').addEventListener('tsChange', (e) => {
  console.log('Value:', e.detail.value);
});
```

```tsx
// React — events are props with on* prefix
<TsButton onTsClick={() => console.log('clicked!')}>Submit</TsButton>
<TsInput onTsInput={(e) => console.log(e.detail.value)} />
```

```vue
<!-- Vue — use @eventName syntax -->
<TsButton @tsClick="handleClick">Submit</TsButton>
<TsInput @tsInput="handleInput" />
```

```html
<!-- Angular — use (eventName) syntax -->
<ts-button (tsClick)="onClick()">Submit</ts-button>
<ts-input (tsInput)="onInput($event)"></ts-input>
```

## CSS Parts

Style internal elements using `::part()`:

```css
ts-button::part(base) {
  text-transform: uppercase;
  letter-spacing: 0.05em;
}

ts-input::part(input) {
  font-size: 1.25rem;
}
```
