## Three-Tier Token Architecture

Tessera UI's design system is built on a strict token hierarchy. Each tier serves a specific purpose, creating a clear path from raw values to themed component styles.

<div class="tier-diagram">
  <div class="tier-card">
    <div class="tier-badge">
      <div class="tier-number" style="background: #1a73e8;">1</div>
      <span class="tier-label">Tier 1</span>
    </div>
    <div>
      <div class="tier-content-title">Reference Tokens</div>
      <div class="tier-content-desc">Raw palette values. Immutable across themes. Never used directly by components.</div>
      <span class="tier-content-example">--ts-ref-primary-600: #1a73e8</span>
    </div>
  </div>
  <div class="tier-card">
    <div class="tier-badge">
      <div class="tier-number" style="background: #0d9488;">2</div>
      <span class="tier-label">Tier 2</span>
    </div>
    <div>
      <div class="tier-content-title">Semantic Tokens</div>
      <div class="tier-content-desc">Purpose-driven aliases to reference tokens. These are what components consume. Remapped in dark mode.</div>
      <span class="tier-content-example">--ts-color-interactive-primary: var(--ts-ref-primary-600)</span>
    </div>
  </div>
  <div class="tier-card">
    <div class="tier-badge">
      <div class="tier-number" style="background: #434751;">3</div>
      <span class="tier-label">Tier 3</span>
    </div>
    <div>
      <div class="tier-content-title">Component Tokens</div>
      <div class="tier-content-desc">Per-component overrides defined on :host. Consumers can customize individual component styles.</div>
      <span class="tier-content-example">--ts-button-bg: var(--ts-color-interactive-primary)</span>
    </div>
  </div>
</div>

This separation means:
- **Themes** only need to remap semantic tokens (Tier 2). Reference tokens stay fixed.
- **Customization** can happen at any level — override a reference token to change the entire palette, a semantic token to adjust one role, or a component token for surgical precision.
- **Dark mode** remaps semantic tokens via `[data-theme="dark"]`. No component CSS changes are needed.

## Global Configuration

Tessera UI uses `data-*` HTML attributes for global configuration — no JavaScript config objects required.

| Attribute | Values | Purpose |
|---|---|---|
| `data-theme` | `light` (default), `dark`, `high-contrast` | Theme selection |
| `data-density` | `comfortable` (default), `compact`, `spacious` | Spacing density |
| `data-icons` | `lucide` (default) | Icon set selection |
| `dir` | `ltr` (default), `rtl` | Text direction |

```html
<!-- Example: dark theme with compact density -->
<html data-theme="dark" data-density="compact">
  <body>
    <ts-button variant="filled">Save Changes</ts-button>
  </body>
</html>
```
